## Card Copy

A data warehouse and Looker Studio dashboard project that turns e-commerce transaction data into star-schema tables, OLAP queries, and business-friendly insights about revenue, profit, sellers, regions, and delivery.

## Full Description

Data Warehouse Shopee is an e-commerce analytics project focused on designing a data warehouse for a Shopee-style business case. The project uses the public Olist dataset, adapted into an Indonesian Shopee context, then processes it through an ETL pipeline using Python and pandas into a SQLite database with a Star Schema approach.

I transformed transaction, product, buyer, seller, date, and shipping data into fact and dimension tables. After that, I explored the data using OLAP queries and visualized the results in Google Looker Studio / Data Studio. The dashboard highlights total revenue, net profit, best-selling categories, regional performance, top sellers, promo-day behavior, and shipping distribution.

## My Part

Designed the data warehouse structure, cleaned and transformed the dataset, built fact and dimension tables, wrote SQL/OLAP analysis, and turned the final result into an interactive Looker Studio dashboard.

## Tiny Proud Detail

This project is where the “messy CSV to decision-ready dashboard” pipeline became very real — less magic, more careful joins.

## Suggested Tags

`Python` `pandas` `SQL` `SQLite` `Star Schema` `Looker Studio`

## Playful Tone Rule

Use playful copy now, but keep it controlled.

Good playful:

- “messy data sits nicely”
- “a tiny analysis studio with slicers”
- “turns rows into a readable story”
- “without making users feel lost in the flow”

Too playful for recruiter-facing sections:

- jokes in every sentence
- exaggerated claims like “mastered everything”
- unclear titles or unclear responsibilities
- hiding important details behind cute wording

Best direction: keep the project facts serious, then add warmth in the descriptions, `my part`, and tiny detail lines.
